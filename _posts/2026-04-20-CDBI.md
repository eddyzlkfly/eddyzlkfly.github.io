## Scenario

After Karen started working for 'TAAUSAI,' she began doing illegal activities inside the company. 'TAAUSAI' hired you as a soc analyst to kick off an investigation on this case.

You acquired a disk image and found that Karen uses Linux OS on her machine. Analyze the disk image of Karen's computer and answer the provided questions.

## Question 1

Which Linux distribution is being used on this machine?

### Scenario

To identify the Linux distribution used on the machine, navigate to the following directory in the disk image:

```bash
/boot
```

This directory typically contains kernel files, initrd images, and configuration files that can reveal the operating system.

Inside the `/boot` directory, there is a file named:

![/assets/img/imgCDBI.png](/assets/img/imgCDBI.png)

The key indicator here is the word **"kali"** in the filename.

## Question 2

What is the MD5 hash of the Apache **access.log** file?

### Solutions

To locate the Apache access log file, navigate to:

```bash
/var/log/apache2/access.log
```

Right-click on `access.log` and select: **Export Hash List**

![/assets/img/imgCDBI(1).png](/assets/img/imgCDBI(1).png)

This will generate hash values (including MD5) for the selected file.

After exporting the hash list, the MD5 value for `access.log` can be found in the generated output.

![/assets/img/imgCDBI(2).png](/assets/img/imgCDBI(2).png)

## Question 7

How many times was Apache run?

### Solutions

To check how many times Apache was run, navigate to:

```bash
/var/log/apache2/access.log
```

This file records incoming HTTP requests handled by the Apache server.

Upon opening the `access.log` file, it is observed that:

![/assets/img/imgCDBI(7).png](/assets/img/imgCDBI(7).png)

- The file is **empty (0 KB)**
- There are **no recorded HTTP requests**

👉 This indicates that the Apache server was **never run**, or at least never handled any requests.

## Question 10

A user executed the **su** command to gain root access multiple times at 11:26. Who was the user?

### Solutions

To identify the user, navigate to:

```bash
/var/log/auth.log
```

This log records authentication attempts, including `su` command usage.

Search for entries around **11:26**, where multiple `su` attempts occurred.

In the log entries at **11:26**, it shows:

![/assets/img/imgCDBI(10).png](/assets/img/imgCDBI(10).png)

## Question 11

### Solutions

To determine the current working directory, navigate to:

```bash

```

From the `.bash_history` file, observe the sequence of `cd` commands. The **last `cd` command** indicates the current working directory.
