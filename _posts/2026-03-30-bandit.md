### **Introduction**

The **OverTheWire Bandit Wargame** is a beginner-friendly cybersecurity challenge designed to teach the fundamentals of Linux, networking, and basic exploitation techniques in a hands-on environment. It is part of the OverTheWire wargames collection, widely used by students, hobbyists, and aspiring penetration testers to build practical skills.

Bandit focuses on guiding players through progressively harder levels, where each stage requires solving a specific problem to retrieve a password for the next level. These challenges introduce essential concepts such as file permissions, command-line navigation, data encoding/decoding, networking protocols, and automation. Instead of relying on theory alone, Bandit emphasizes **learning by doing**, making it an excellent starting point for anyone new to cybersecurity or ethical hacking.

In this writeup, each level is explained step-by-step, showing the commands used, the reasoning behind them, and the key takeaway from each challenge. By following along, you’ll not only learn how to complete the Bandit wargame but also gain a solid foundation in real-world Linux and security concepts.

### **Bandit 01**

The password for the next level is stored in a file called **-** located in the home directory.

``` shell 
bandit1@bandit:~$ ls -la

-rw-r--r--   1 root    root    3851 Oct 14 09:19 .bashrc

263JGJPfgU6LtdEvgfWU1XP5yac29mFx
```

> Solution: Since the shell or the program (like cat) defaults to standard input (stdin) when it sees a single -, providing the path ./- is the most common way to "break" that logic and force it to see the actual file.

### **Bandit 06**
Find a file somewhere on the server. Properties:

- owned by user bandit7
- owned by group bandit6
- 33 bytes in size

``` shell 
bandit6@bandit:~$ find / -type f -size 33c -group bandit6 -user bandit7 2>&1 | grep -v "Permission denied"
find: ‘/proc/26/task/26/fdinfo/6’: No such file or directory

morbNTDkSW6jIlUc0ymOdMaLnOlFVAaj
```

> Solution: You used the ``find /`` command to search the entire system for a specific file. By adding ``-type f``, ``-size 33c``, ``-group bandit6``, and ``-user bandit7``, you filtered the results to match the exact properties of the password file.
The part ``2>&1 | grep -v "Permission denied"`` is a clever trick to hide all the "Permission denied" errors that pop up when searching folders you don't have access to, leaving only the correct file path: ``/var/lib/dpkg/info/bandit7.password.``

### **Bandit 14**
The password for the next level can be retrieved by submitting the password of the current level to port **30000 on localhost**.

``` shell
root@kali:~# ssh -i sshkey.private bandit14@bandit.labs.overthewire.org -p 2220
 
bandit14@bandit:~$ cat /etc/bandit_pass/bandit14 | nc localhost 30000
Correct!
8xCjnmgoKbGLhHFAZlGE5Tmu4M2tKJQo
bandit15@bandit:~$
```

> You used ``cat`` to read your current password and ``nc`` (netcat) to "send" it to ``localhost`` on port ``30000``. The server checked your password, saw it was correct, and sent back the password for the next level.
